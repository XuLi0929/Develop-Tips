Preventing Data Loss: Automated Backups for Your Working Directory

Published: February 10, 2026 · 3-part guide · Updated: February 12, 2026

Part 1 of 3 — Create the Backup Script

If you're working on a remote server and worry about losing files due to accidental deletion or a crash, a quick local backup script can save you a lot of headaches. In this three-part guide, we'll walk you through setting up a fully automated, persistent backup of your current working directory.

Let's start by creating the core backup script.

Create a bash script that zips your working directory

Create a file called backup.sh that zips the entire current directory into a single archive called backup_v1.zip. This gives you a complete snapshot you can restore from if anything goes wrong.

#!/bin/bash
zip -r backup_v1.zip .

Save this as backup.sh in your working directory and make it executable:

chmod +x backup.sh
